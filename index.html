Distributed Data Frame

Target Audience: Data Scientists & Engineers

Goal: To Maximize Your Data Analysis Productivity through Simplicity, without compromising Power & Sophistication.

    DDF (Distributed DataFrame) provides a powerful yet simple, table-like abstraction on top of any Big-Data engine underneath.

    DDF has been implemented on top of Apache Spark, and is the engine powering Adatao's data applications.

  • Treat parallel, distributed data sets like one big table
  • Idiomatic R-inspired user experience
  • Focus on analytics, not MapReduce
  • Seamlessly integrate with external ML libraries
  • Share and collaborate on DDFs via URIs
  • Use multiple languages (Java, Scala, R, Python)

//To start working with a DDF-on-Spark cluster:
DDFMananager smanager = DDFManager.get("spark");

//Then, data can be loaded into a SparkDDF as follows:
DDF table = smanager.sql2ddf("select * from airline");

/* Basic Statistic */
Summary[] summary = table.summary()

/* ETL, transform */
table = table.Transform.transformUDF("dist= round(distance/2, 2)");

DDF Principles

The ease of app development on RDBMS

The SQL abstraction has boosted app developer productivity tremendously, hiding away all the complexity and diversity of the database engines underneath.

The sophistication of R

For decades, data analysis idioms and packages have evolved around the powerful concept of the data.frame, from basic data transformation, filtering and projection, to advanced data mining and machine learning.

The scale of parallel, distributed computing

Thanks to technologies like Hadoop MapReduce, Apache Spark, and other parallel computing frameworks, big compute capabilities have become widely available.
